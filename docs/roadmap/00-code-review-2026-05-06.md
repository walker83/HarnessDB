> 共发现 ~80+ 个问题

## 严重程度分布

| 严重程度 | 数量 | 说明 |
|---------|------|------|
| **Critical** | ~22 | 数据丢失、错误结果、系统不可用 |

## 一、存储引擎层 (be-storage / be-segment)

### Critical

#### 1. 无 WAL，崩溃即丢数据
- **位置**: `be-storage/src/engine.rs:24-39`
- **问题**: 写入只进内存 memtable，无预写日志。进程崩溃后所有未刷盘数据永久丢失。
- **修复**: 实现 WAL，先写日志再写内存，启动时回放。

#### 2. 启动不恢复数据
- **位置**: `be-storage/src/engine.rs:26-39`
- **问题**: `StorageEngine::open()` 每次创建空引擎，不扫描磁盘加载已有的 tablet/rowset 元数据。重启后数据不可见。
- **修复**: 启动时扫描 `data_dir`，加载 `rowset_*.json`，重建内存状态。

#### 3. ZoneMap 索引完全失效
- **位置**: `be-storage/src/index.rs:69-71`
- **问题**: `deserialize_scalar()` 总是返回 `Binary(...)`，`compare_scalars` 走 catch-all 返回 `Equal`。所有 ZoneMap 裁剪等于没用，每次全量扫描。
- **修复**: 按列的类型信息反序列化 min/max 值。

#### 4. MemTable::to_block() 丢行
- **位置**: `be-storage/src/tablet.rs:143-220`
- **问题**: 用 `filter_map` 构建列向量，遇到 Null 值直接跳过。不同列长度不一致，产生损坏的 Block。
- **修复**: 使用 `is_null()` 检查，对 null 值 push `None`。

#### 5. DELETE 只删 memtable，刷盘后数据复活
- **位置**: `be-storage/src/tablet.rs:353-366`
- **问题**: 删除只操作内存，已刷盘的 segment 文件中数据不受影响。flush 后"已删除"数据重新出现。
- **修复**: 实现 delete bitmap / tombstone 机制，compaction 时物理清理。

#### 6. Flush 不原子
- **位置**: `be-storage/src/tablet.rs:369-417`
- **问题**: 写 segment → 写元数据 → 加内存列表三步非原子。崩溃后留下孤儿文件或丢失元数据。
- **修复**: 先写临时文件，fsync，再 rename。

#### 7. Compaction 竞争条件 + 磁盘泄漏
- **位置**: `be-storage/src/engine.rs:150-168`
- **问题**: `remove_rowsets` 和 `add_rowset` 之间有窗口期，并发读看到零 rowset。且 compaction 后旧 segment 文件永不删除。
- **修复**: 原子交换 rowset 列表；compaction 后删除旧文件。

#### 8. Auto-Flush 竞争
- **位置**: `be-storage/src/tablet.rs:333-349`
- **问题**: 写锁释放和 flush 重获锁之间，并发写入可能丢失或重复 flush。
- **修复**: 原子交换 memtable（换新空 memtable，旧的脱离锁 flush）。

#### 9. Segment 读错误被静默吞掉
- **位置**: `be-storage/src/tablet.rs:444-453`
- **问题**: segment 读取失败时 log warning 然后 skip，查询返回不完整结果无报错。
- **修复**: 向上传播错误。

#### 10. 解压失败静默返回损坏数据
- **位置**: `be-segment/src/codec.rs:104-121`
- **问题**: 解压失败返回原始压缩字节，调用方无法区分有效数据和损坏数据。
- **修复**: 返回 `Result`，不静默 fallback。

### High

#### 11. Segment 无 CRC 校验
- **位置**: `be-storage/src/segment/writer.rs:82-137`
- **问题**: segment 文件无 checksum，静默磁盘损坏产生错误结果无法检测。

#### 12. drop_tablet 非原子
- **位置**: `be-storage/src/engine.rs:62-76`
- **问题**: 先删内存后删磁盘，中间崩溃导致孤儿数据。

#### 13. 编码类型声称但不执行
- **位置**: `be-storage/src/segment/writer.rs:217-231`
- **问题**: `choose_encoding` 返回 BitPacked/Dictionary/RLE 但 `write_page` 只处理 Lz4/Zstd，其他类型存原始数据。

#### 14. Null vs 空字符串歧义
- **位置**: `be-storage/src/segment/writer.rs:313-331`
- **问题**: null 字符串编码为 `len=0`，与合法空字符串 `""` 相同。反序列化时 null 变成 `Some("")`。

#### 15. 整个 segment 文件读入内存
- **位置**: `be-storage/src/segment/reader.rs:50-51`
- **问题**: `std::fs::read(path)` 加载完整 segment，大文件导致 OOM。

#### 16. Compaction 不去重
- **位置**: `be-storage/src/compaction.rs:100-162`
- **问题**: 简单 `Block::concat`，对 key 列不去重，违反唯一性约束。

#### 17. Compaction 全量加载内存
- **位置**: `be-storage/src/compaction.rs:109-121`
- **问题**: 所有待合并 segment 全部读入内存，大 tablet 可耗尽内存。

### Medium

#### 18. Footer 用 JSON 序列化
- **位置**: `be-storage/src/segment/writer.rs:123-124`
- **问题**: 大宽表 footer 极大，JSON 解析慢。建议用二进制格式。

#### 19. LZ4 解压 fallback 不可靠
- **位置**: `be-storage/src/codec.rs:79-85`
- **问题**: 依赖 size 启发式判断是否压缩，可能误判。

#### 20. Memtable 行存格式内存开销大
- **位置**: `be-storage/src/tablet.rs:57-131`
- **问题**: 每行独立 `Vec<ScalarValue>` 堆分配，cache 局部性差。

#### 21. 无全局内存预算
- **位置**: `be-storage/src/tablet.rs:318-329`
- **问题**: 每个 tablet 64MB memtable，N 个 tablet 总内存 N×64MB 无上限。

## 二、查询引擎层 (fe-sql-parser / fe-sql-planner / fe-expression)

### Critical

#### 22. NULL 不传播
- **位置**: `fe-expression/src/evaluator.rs:165-172`
- **问题**: 算术/比较/逻辑运算不检查 null bitmap。`NULL + 5` 应返回 `NULL`，实际返回错误值。

#### 23. CaseWhen/coalesce/nullif 返回 1 元素向量
- **位置**: `fe-expression/src/evaluator.rs:103`、`fe-expression/src/functions.rs:231,245`
- **问题**: `result.into_iter().next().map(|v| Vector::from_scalar(&v, 1))` 只取第一个元素。

#### 24. SemiJoin key 推断错误
- **位置**: `fe-sql-planner/src/planner.rs:651-666`
- **问题**: EXISTS 子查询用 `output_columns().first()` 作为 join key，与实际关联条件无关。

#### 25. SQL 注入
- **位置**: `fe-sql-planner/src/planner.rs:176-177`
- **问题**: DESCRIBE/SHOW COLUMNS 直接 format! 拼接用户输入。

#### 26. Set 操作被丢弃
- **位置**: `fe-sql-parser/src/parser.rs:1463-1494`
- **问题**: UNION ALL/EXCEPT/INTERSECT 被直接 `let _ = (...)` 忽略。

#### 27. 未处理表达式静默变 Wildcard
- **位置**: `fe-sql-parser/src/parser.rs:1774`
- **问题**: CAST、CASE、Like 等不识别的表达式变成 `Wildcard`。

#### 28. 未知运算符默认为 Eq
- **位置**: `fe-sql-parser/src/parser.rs:1794`
- **问题**: BitwiseAnd、Glob 等不识别运算符静默变 `=`。

#### 29. 物化视图重写忽略 WHERE
- **位置**: `fe-sql-planner/src/materialized_view.rs:139-153`
- **问题**: MV rewrite 只做简单 scan，丢弃原始查询的 WHERE/GROUP BY。

#### 30. Runtime Filter 永远不生效
- **位置**: `fe-sql-planner/src/runtime_filter.rs:39`
- **问题**: 只匹配 `HashJoin` 节点，但 planner 生成的是 `Join` 节点。

### High

#### 31. CTE 规划错误
- **位置**: `fe-sql-planner/src/planner.rs:477-507`
- **问题**: CTE 包装后主查询无法引用 CTE 名称。

#### 32. 谓词下推不处理 JOIN
- **位置**: `fe-sql-planner/src/optimizer.rs:50-82`
- **问题**: 谓词只推过 Scan/Filter/Project，JOIN 上方谓词不推入。

#### 33. 谓词下推不处理列重命名
- **位置**: `fe-sql-planner/src/optimizer.rs:58-65`
- **问题**: 推过 Project 时不重映射列引用，别名谓词失败。

#### 34. Cost 模型 join selectivity 硬编码 0.01
- **位置**: `fe-sql-planner/src/cost_model.rs:237`
- **问题**: 不考虑 key cardinality，主键 join selectivity 估算严重偏离。

#### 35. 整数除零 panic
- **位置**: `fe-expression/src/evaluator.rs:114`
- **问题**: `a / b` 在 b=0 时 Rust panic。

#### 36. 混合类型算术返回 0
- **位置**: `fe-expression/src/evaluator.rs:165-172`
- **问题**: Int64 + Float64 走默认分支返回零向量。

#### 37. 日期语义不一致
- **位置**: `fe-expression/src/functions.rs:596-601`
- **问题**: `naive_date_from_ordinal` 把存储值当作年天数应用到 1970 年。

#### 38. CTE 解析只取第一个
- **位置**: `fe-sql-parser/src/parser.rs:1439`
- **问题**: `w.cte_tables.first()` 丢失后续 CTE。

#### 39. distinct 标志丢失
- **位置**: `fe-sql-parser/src/parser.rs:1708`
- **问题**: `COUNT(DISTINCT x)` 和 `COUNT(x)` 不区分。

### Medium

#### 40. SELECT DISTINCT 未处理
- **问题**: Parser 不区分 `SELECT DISTINCT a` 和 `SELECT a`。

#### 41. 无窗口函数 plan node
- **问题**: fe-expression 实现了 row_number/rank/lag/lead，但 planner 无 Window plan node。

#### 42. 无多表 join 排序优化
- **问题**: `reorder_joins` 只考虑二元 join，不搜索全局最优序。

## 三、执行引擎层 (be-execution)

### Critical

#### 43. FilterExecNode 是空操作
- **位置**: `be-execution/src/exec_node.rs:569-583`
- **问题**: 创建全 true bitmap 然后过滤，predicate 从未解析求值。WHERE 子句无效。

#### 44. ProjectExecNode 是直通
- **位置**: `be-execution/src/exec_node.rs:610-612`
- **问题**: 直接委托 child.get_next()，不执行列投影或表达式计算。

#### 45. Planner 传空参数给 Agg/Sort/Join
- **位置**: `be-execution/src/planner.rs:113-161`
- **问题**: `group_by: vec![]`, `aggregates: vec![]`, `order_by: vec![]`。

#### 46. 日期解析完全错误
- **位置**: `be-execution/src/predicate_parser.rs:280-284`
- **问题**: `"2024-01-15".replace('-', "")` → `20240115`，不是 epoch 天数。

#### 47. 日期序列化硬编码 1970-01-01
- **位置**: `mysql-protocol/src/packet.rs:691-695`
- **问题**: Date/DateTime 输出忽略实际值。

#### 48. 聚合只处理 Int64
- **位置**: `be-execution/src/exec_node.rs:645-692`
- **问题**: Float/Int32 等类型返回 Null 或 0。

#### 49. 协调器是模拟的
- **位置**: `fe-scheduler/src/coordinator.rs:227-281`
- **问题**: 不发 RPC，直接返回空结果。

#### 50. BE gRPC 服务是空壳
- **位置**: `rpc/src/be_service.rs:39-62`
- **问题**: exec_plan_fragment 标记 Running→Completed 不执行。fetch_data 返回空。

#### 51. FE 不走分布式路径
- **位置**: `roris-server/src/fe_main.rs:1084-1131`
- **问题**: INSERT/SELECT 直接本地执行，跳过分布式调度。

#### 52. 无表达式求值引擎
- **问题**: 整个 be-execution 缺少表达式求值器，Filter/Project/JOIN 条件都存为字符串未执行。

#### 53. StreamLoad 解析完就丢
- **位置**: `data-io/src/stream_load.rs:99-155`
- **问题**: 解析 CSV/JSON 后 block 被直接丢弃，没有写入存储。

#### 54. 错误被静默吞掉
- **位置**: `be-execution/src/planner.rs:370-373`
- **问题**: 执行错误 log 然后 break，返回部分结果，调用方无法区分完整和截断。

### High

#### 55. LEFT JOIN 未实现
- **位置**: `be-execution/src/exec_node.rs:1236-1238`
- **问题**: 代码块为空。

#### 56. Window 聚合算全列
- **位置**: `be-execution/src/exec_node.rs:1629-1644`
- **问题**: 不按分区范围切片，PARTITION BY 无效。

#### 57. Sort O(n²) 构建
- **位置**: `be-execution/src/exec_node.rs:927-935`
- **问题**: 逐行 slice + append，大结果集极慢。

#### 58. Sort/Agg/Window 全量物化
- **位置**: `exec_node.rs:882-899`, `exec_node.rs:709-712`
- **问题**: 无 spill-to-disk，大表导致 OOM。

#### 59. Scan 错误返回空 block
- **位置**: `be-execution/src/exec_node.rs:493-499`
- **问题**: storage 读取失败时 log warn 返回空，查询结果错误。

#### 60. Pipeline::execute 无资源清理

#### 61. Insert 列重排逻辑错误
- **位置**: `be-execution/src/exec_node.rs:288-310`

#### 62. HashPartition 全互联

#### 63. Fragment 并行度硬编码 1

## 四、分布式 & 基础设施

### High

#### 69. Table ID 全是 0
- **位置**: `roris-server/src/fe_main.rs:499`
- **问题**: 所有表共享 ID=0，tablet 映射全部冲突。

#### 70. Catalog TOCTOU 竞争
- **位置**: `roris-server/src/fe_main.rs:45`
- **问题**: 外层 StdRwLock + 内层 DashMap 双重锁，create_table 先读锁再写锁有窗口。

#### 71. Catalog 写不原子
- **位置**: `fe-catalog/src/catalog.rs:132-146`
- **问题**: `fs::write` 中途崩溃损坏文件。

#### 72. fe_main 绕过 CatalogWriter
- **位置**: `roris-server/src/fe_main.rs:467-527`
- **问题**: 直接操作 CatalogManager，edit log 不更新，重启丢元数据。

#### 73. FE 每次 INSERT 新建 CatalogManager
- **位置**: `roris-server/src/fe_main.rs:1100-1108`
- **问题**: 不共享 catalog 状态。

#### 74. MemoryTracker TOCTOU 竞争
- **位置**: `fe-scheduler/src/memory.rs:21-33`
- **问题**: load + store 非原子，并发分配可超额。

#### 75. Update 非原子
- **位置**: `be-execution/src/exec_node.rs:1825-1828`

#### 76. Transaction 部分提交
- **位置**: `roris-server/src/fe_main.rs:129-148`
- **问题**: 顺序 apply，中间失败不回滚。

#### 77. NULL 等于 NULL
- **位置**: `types/src/runtime_filter.rs:152-155`

#### 78. Vector::append_vector 丢弃 String/Json/Null 类型
- **位置**: `types/src/vector.rs:1192-1206`
- **问题**: `_ => {}` 静默忽略字符串等类型，Block::concat 丢列。
