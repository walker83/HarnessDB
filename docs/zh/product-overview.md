# RorisDB 产品概述

## 什么是 RorisDB

RorisDB 是一个使用 Rust 重新实现的**实时 OLAP（在线分析处理）数据库**，架构上受到 Apache Doris 的启发。

**RorisDB** = **R**ust + (D)**oris** + **DB**

### 项目定位

RorisDB 不是一个 Apache Doris 的 fork 或包装器，而是一个独立重新实现的开源项目。它采用 Rust 语言重写，目标是获得：

- **内存安全性**：通过 Rust 的所有权系统防止内存安全漏洞
- **零成本抽象**：高性能的同时保持代码可维护性
- **细粒度资源控制**：基于 Tokio 的异步 I/O 和可配置的内存限制

### 核心特性

- **实时分析**：支持实时数据导入和秒级查询响应
- **列式存储**：采用列式内存布局，优化分析查询性能
- **向量化执行**：批量处理数据，充分利用 CPU 缓存和 SIMD 指令
- **分布式架构**：MPP（大规模并行处理）架构，支持水平扩展
- **MySQL 兼容**：支持 MySQL 协议，可直接使用 MySQL 客户端连接
- **丰富的数据类型**：支持数值、字符串、日期等多种数据类型
- **完整的 SQL 支持**：支持复杂查询、聚合、窗口函数等

## 为什么选择 RorisDB

### 与 Apache Doris 的关系

RorisDB 在架构上借鉴了 Apache Doris 经过验证的设计理念：

- 相同的 MPP 架构
- 类似的列式存储格式（Tablet → Rowset → Segment）
- 相同的 Compaction 策略（Cumulative + Base）

但 RorisDB 是一个完全独立的实现，使用 Rust 语言重写，带来以下优势：

| 对比维度 | Apache Doris (C++) | RorisDB (Rust) |
|---------|-------------------|----------------|
| 内存安全 | 需要谨慎管理 | 编译期保证 |
| 并发安全 | 依赖开发者经验 | 借用检查器保证 |
| 云原生部署 | 支持 | 无不安全代码路径 |
| 资源管理 | 手动控制 | 细粒度控制 |

### 适用场景

RorisDB 适用于以下场景：

- **实时数据分析**：需要快速分析最新数据的业务
- **数据仓库**：构建企业级数据仓库
- **BI 报表**：支持商业智能工具的即席查询
- **用户行为分析**：分析用户行为、点击流等数据
- **日志分析**：实时分析系统日志、应用日志

## 技术架构

RorisDB 采用经典的 FE（Frontend）+ BE（Backend）架构：

```
┌─────────────────────────────────────────────────────────────────┐
│                        RorisDB Cluster                            │
│                                                                 │
│   ┌──────────────┐                        ┌──────────────────┐    │
│   │  FE (Rust)  │◄────── RPC ─────────►│   BE 1 (Rust)   │    │
│   │              │     (gRPC)            │                  │    │
│   │  ┌────────┐ │                        │  ┌────────────┐  │    │
│   │  │ Parser │ │                        │  │ Storage   │  │    │
│   │  │Planner │ │                        │  │ Engine    │  │    │
│   │  │Scheduler│ │                        │  │ (Segment) │  │    │
│   │  │Catalog │ │                        │  └────────────┘  │    │
│   │  └────────┘ │                        │  ┌────────────┐  │    │
│   │              │                        │  │ Execution  │  │    │
│   │  MySQL      │                        │  │ Pipeline   │  │    │
│   │  Protocol   │                        │  └────────────┘  │    │
│   └──────────────┘                        └──────────────────┘    │
│          │                                        │               │
│          │             ┌──────────────────┐    │               │
│          └───────────►│   BE 2 (Rust)   │◄───┘               │
│                         └──────────────────┘                   │
└─────────────────────────────────────────────────────────────────┘
```

### FE（Frontend）职责

- SQL 解析和语法分析
- 查询规划和优化
- 元数据管理（数据库、表、分区）
- 查询调度和协调
- 集群管理（BE 节点注册、心跳）

### BE（Backend）职责

- 数据存储和管理（Tablet、Rowset、Segment）
- 查询执行（Pipeline 执行引擎）
- 数据压缩和 Compaction
- 数据导入和导出

## 当前状态

RorisDB 目前处于 **Proof-of-Concept** 阶段（v0.1.3），核心功能已基本完成，包括：

- ✅ SQL 解析和查询规划
- ✅ 向量化表达式引擎
- ✅ 聚合函数和窗口函数
- ✅ 列式存储引擎
- ✅ MySQL 协议支持
- ✅ 分布式查询调度
- ✅ 数据导入（CSV、JSON）

正在进行的功能：

- 🚧 物化视图
